# Contributing to GeoAware Bible

Thank you for contributing to GeoAware Bible. This project exists to help pilgrims walk through Scripture with reverence, clarity, and geographic awareness.

Before contributing, ask:

> Does this help the pilgrim walk the Word?

## Mission Alignment

Every contribution should align with the mission of walking pilgrims through Scripture. Features, fixes, documentation, and design changes should keep Scripture as the primary experience and technology as the servant.

Avoid changes that introduce feature bloat, addictive mechanics, social-media imitation, or visual spectacle that distracts from Scripture.

## Accessibility

Accessibility is mandatory. Contributions should support readable text, keyboard access where applicable, semantic structure, sufficient contrast, and inclusive interaction patterns.

A pilgrim should not be excluded from the journey because of avoidable design or engineering choices.

## Performance

Performance comes before visual effects. Maps, animations, media, and AI-assisted features should be designed with responsiveness, loading behavior, offline practicality, and device constraints in mind.

Do not make the experience heavier without a clear architectural purpose.

## Maintainability

Write code and documentation that future contributors can understand. Prefer clear names, focused modules, and simple architecture over cleverness.

Every commit should have a clear architectural purpose.

## Respectful Language

Use respectful, humble, and professional language in code, documentation, issues, commits, and reviews. GeoAware Bible serves pilgrims from many backgrounds and languages.

## Clean Commits

Keep commits focused and understandable. Use commit messages that clearly describe the change. Do not mix unrelated refactors, formatting, dependency changes, or feature work into the same commit.

## Scripture Licensing

Do not add copyrighted Bible text without a verified license. If Scripture text is introduced, the source, license, and usage rights must be documented before inclusion.

When in doubt, do not add the text.
